## 四、问题修复记录

### 4.1 JT809Encoder ByteBuf 引用计数问题

**问题描述**:  
编码器在处理 ByteBuf 时存在内存泄漏风险。

**修复方案**:
- 使用 try-finally 确保资源正确释放
- 新增 `calculateCrc()` 方法避免创建临时 ByteBuf

**修复文件**: `jt809-core/src/main/java/org/iot/v/jt809/core/codec/JT809Encoder.java`

### 4.2 BCDUtilTest 奇数长度字符串编码预期值错误

**修复方案**: 修正预期值：`"12345"` → `0x12 0x34 0x50`

### 4.3 SessionTest 无参构造函数检查错误

**修复方案**: 移除 `assertNotNull(s.getSessionId())` 断言

### 4.4 SessionManagerTest 会话 ID 唯一性问题

**修复方案**: 添加 `@AfterEach` 清理方法，使用 `UUID.randomUUID()` 确保唯一性

### 4.5 M1EncryptUtilTest 参数顺序错误

**修复方案**: 修正 `encrypt()` 方法参数顺序为 `(byte[], long, String)`

### 4.6 JT809DecoderTest SequenceGenerator 方法调用错误

**修复方案**: 改为 `SequenceGenerator.getInstance().next()`

### 4.7 JT809Client 缺少 isConnected 方法

**修复方案**: 添加 `isConnected()` 方法返回连接状态

### 4.8 消息头长度常量错误 ✅ 已修复

**问题描述**:  
服务端解码消息时出现长度不匹配警告
```
Message length mismatch: declared=136, actual=143
```

**根本原因**:  
`MESSAGE_HEAD_LENGTH` 常量值为 22，但消息头实际长度为 29 字节：
- 消息长度：4字节
- 消息流水号：4字节
- 消息ID：2字节
- 平台接入码：8字节
- 版本号：2字节
- 上级平台接入码：4字节
- 加密方式：1字节
- 加密密钥：4字节

**修复方案**:  
修改 `JT809Constant.MESSAGE_HEAD_LENGTH = 29`

**修复文件**: `jt809-core/src/main/java/org/iot/v/jt809/core/constant/JT809Constant.java`

### 4.9 CRC 校验位置错误 ✅ 已修复

**问题描述**:  
消息长度匹配后，CRC 校验失败
```
CRC check failed: calculated=0x7882, received=0x0
```

**根本原因**:  
解码器中 CRC 读取位置计算错误

**修复方案**:  
修正 CRC 数据切片长度计算：
```java
int messageDataLength = declaredLength - JT809Constant.CRC_LENGTH;
ByteBuf messageData = originalMessage.slice(0, messageDataLength);
```

**修复文件**: `jt809-core/src/main/java/org/iot/v/jt809/core/codec/JT809Decoder.java`

### 4.10 客户端关闭时触发重连 ✅ 已修复

**问题描述**:  
客户端调用 `stop()` 关闭时，仍会触发重连任务

**根本原因**:  
`ReconnectHandler.channelInactive()` 未检查客户端运行状态

**修复方案**:  
在 `channelInactive()` 和 `scheduleReconnect()` 中增加运行状态检查：
```java
if (!client.isRunning()) {
    log.info("Client is shutting down, skip reconnect");
    return;
}
```

**修复文件**: 
- `jt809-client/src/main/java/org/iot/v/jt809/client/handler/ReconnectHandler.java`
- `jt809-client/src/main/java/org/iot/v/jt809/client/JT809Client.java`

## 五、新增功能

### 5.1 ServerHandler 消息监听器

**功能描述**:  
服务端处理器增加消息监听器机制，支持外部监听业务消息

**新增接口**:
```java
@FunctionalInterface
public interface MessageListener {
    void onMessage(Session session, BaseMessage message);
}
```

**新增方法**:
- `addMessageListener(MessageListener listener)` - 添加监听器
- `removeMessageListener(MessageListener listener)` - 移除监听器

**文件**: `jt809-server/src/main/java/org/iot/v/jt809/server/handler/ServerHandler.java`
